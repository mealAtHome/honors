아주 똑똑하구나.

지금은 라인업 템플릿 기능을 추가하고 있다.
가장 최근 커밋 내용을 보면 대충 알 수 있다.
아래 내용을 전부 읽고 작업을 진행하도록 해라.
작업이 끝나면 커밋하지 말고 언스테이지 상태 그대로 두도록 해라.

F10-ClassUpdate011Lineuptmp.html : 라인업의 템플릿을 저장, 불러오기, 삭제, 이름변경을 진행하는 메인 페이지이다.
F10-ClassUpdate010LineupUpdate.html 에서 #CUTF-btn-lineuptmp 를 클릭하면 넘어가는 페이지이다.

1. 내가 honors-sql/ 안에 오늘날짜로 스키마 파일을 넣어두었다. clslineuptmpa, clslineuptmpb 두 테이블을 최근 추가하였고, 부차적인 작업은 아직 진행을 못한 상태이다. 테이블을 추가함에 따라, 필요한 BO, model, api 등 파일들을 추가해달라.

2. 해당 페이지에서 세 개의 탭이 있다.

3. "저장" 텝에서 현재 입력한 라인업을 저장하는 기능을 구현하여라.

4. "템플릿" 탭에서 저장한 템플릿을 불러오고, 삭제, 이름을 변경할 수 있는 기능, 그리고 상세를 볼 수 있는 기능을 추가하여라. 페이지네이션은 50개 단위로 적용해라.
4-1. 상세를 보는 페이지는 추가하지 말고, 그 해당 페이지 안에서 볼 수 있도록 구현하여라.
4-2. 상세보기 버튼을 클릭하면, 리스트 페이지는 숨기고, 상세 div 를 표시하여 보일 수 있도록 해라.
4-3. 상세를 클릭하였을 때, ajax 통신을 통해서, clslineuptmpb 리스트를 조회해서, 표현할 수 있도록 해라.
4-4. 상세에서 돌아가는 버튼을 클릭하면, 최근 보고있던 페이지네이션번호를 기억해서 리스트를 표시하도록 하여라. 숨겼다가 다시 표시해도 상관없다.

5. "최근일정" 탭을 클릭하면 현재의 cls를 제외하고 최신 cls 순으로 작성한 라인업 리스트를 표시하도록 하여라.
라인업 리스트를 표시할 때, 각 라인업의 상세보기 버튼을 추가하여, 상세보기 기능을 구현하여라. 페이지네이션은 50개 단위로 적용해라.
그리고 유저가 clslineupa 전체를 가져올지, clslineupa 하나를 가져올 지 선택할 수 있는 버튼을 구현하여라.

6. "저장" 탭에서 "라인업 확인"이라고 하는 항목이 있는데, 페이지를 처음 켰을 때, CUTF 페이지에서 유저가 입력한 라인업을 복사하여 가져오도록 되어있다.
6-1. 여기서 "삭제" 버튼을 추가해서 유저가 원하는 템플릿만 저장할 수 있도록 구현하여라.

7. "템플릿"이나 "최근일정"에서 라인업 템플릿을 가져와서 "참가설정"에 적용하는 기능을 구현하여라. 단, 기존 내용을 변경하는 것이 아닌, 새로운 라인업을 추가하는 형태로 구현하여라.
